# Cactus CQ Pipeline

Convert Hugging Face-style checkpoints into Cactus-compatible CQ `.weights`
directories, then optionally QDQ them back into a PyTorch/HF-style checkpoint for
validation.

Run commands from `/workspace`:

```bash
export PYTHONPATH=/workspace
```

Generated artifacts should go under `/workspace/cactus_cq_runs`, not inside this
source tree.

## 1. Prepare Calibration Data

The converter expects a calibration manifest. For best results, use trajectories
generated by the same model family and modality.

Manifest shape:

```json
{
  "language": {
    "path": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/language.jsonl"
  },
  "vision": {
    "path": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/vision.jsonl",
    "base_dir": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL"
  },
  "audio": {
    "path": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/audio.jsonl",
    "base_dir": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL"
  },
  "transcription": {
    "path": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/transcription.jsonl",
    "base_dir": "/workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL"
  }
}
```

Include only the sections the model actually supports. A text-only model should
not use a `vision` section.

Common row schemas:

- Language: `{"prompt_text": "...", "completion_text": "..."}`
- Language chat: `{"messages": [{"role": "user", "content": "..."}]}`
- Vision: `{"image_path": "images/example.jpg", "prompt_text": "...", "completion_text": "..."}`
- Audio instruction: `{"audio_path": "audios/example.wav", "prompt_text": "...", "completion_text": "..."}`
- Transcription: `{"audio_path": "audios/example.wav", "transcript": "..."}`

## 2. First Full Conversion

Use the highest desired precision first and save a Hessian cache. Later lower-bit
variants can reuse this cache and skip calibration forwards.

### Text-Only Model

Example: language linears CQ4, embeddings CQ4.

```bash
python -m src.cq_convert convert \
  --model /path/to/hf_model \
  --out /workspace/cactus_cq_runs/MODEL_L4 \
  --bits 4 \
  --language-bits 4 \
  --calibration-manifest /workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/calibration_manifest.json \
  --device cuda \
  --model-family auto \
  --max-language-examples 1500 \
  --hessian-gpu-flush-interval 16 \
  --hessian-progress \
  --hessian-cache-out /workspace/cactus_cq_runs/hessian_cache/MODEL_native \
  --force
```

### Vision-Language Model

Example: language linears CQ4, vision linears CQ4.

```bash
python -m src.cq_convert convert \
  --model /path/to/hf_vlm \
  --out /workspace/cactus_cq_runs/MODEL_L4V4 \
  --bits 4 \
  --language-bits 4 \
  --vision-bits 4 \
  --calibration-manifest /workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/calibration_manifest.json \
  --device cuda \
  --model-family auto \
  --max-language-examples 1500 \
  --max-vision-examples 1500 \
  --calibration-batch-size 1 \
  --hessian-gpu-flush-interval 16 \
  --hessian-progress \
  --hessian-cache-out /workspace/cactus_cq_runs/hessian_cache/MODEL_native \
  --force
```

If GPU memory is tight, lower `--hessian-gpu-flush-interval` to `1`, `2`, or
`4`.

### Audio Or Transcription Model

Audio/transcription weights use CQ without GPTQ correction.

```bash
python -m src.cq_convert convert \
  --model /path/to/hf_audio_model \
  --out /workspace/cactus_cq_runs/MODEL_L4 \
  --bits 4 \
  --audio-bits 4 \
  --calibration-manifest /workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/calibration_manifest.json \
  --device cuda \
  --model-family auto \
  --max-audio-examples 1500 \
  --max-transcription-examples 1500 \
  --hessian-progress \
  --force
```

Use the relevant limit for the manifest section you have.

## 3. Build Lower Bit Variants From Cache

Text-only lower-bit variants:

```bash
for bits in 1 2 3; do
  python -m src.cq_convert convert \
    --model /path/to/hf_model \
    --out /workspace/cactus_cq_runs/MODEL_L${bits} \
    --bits ${bits} \
    --language-bits ${bits} \
    --calibration-manifest /workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/calibration_manifest.json \
    --device cuda \
    --model-family auto \
    --hessian-cache-in /workspace/cactus_cq_runs/hessian_cache/MODEL_native \
    --force
done
```

VLM lower-language-bit variants with vision fixed at CQ4:

```bash
for bits in 1 2 3; do
  python -m src.cq_convert convert \
    --model /path/to/hf_vlm \
    --out /workspace/cactus_cq_runs/MODEL_L${bits}V4 \
    --bits 4 \
    --language-bits ${bits} \
    --vision-bits 4 \
    --calibration-manifest /workspace/cactus_cq_runs/native_trajectories_1500_min257/MODEL/calibration_manifest.json \
    --device cuda \
    --model-family auto \
    --hessian-cache-in /workspace/cactus_cq_runs/hessian_cache/MODEL_native \
    --force
done
```

Naming convention:

- Text-only: `L1`, `L2`, `L3`, `L4`.
- VLM with vision CQ4: `L1V4`, `L2V4`, `L3V4`, `L4V4`.

## 4. Inspect The Conversion

Every output directory should contain:

- `.weights` files
- `config.txt`
- tokenizer/runtime files
- `conversion_manifest.json`
- `conversion_summary.json`
- `hessian_metadata.json`
- optionally `hessians.safetensors`

Quick checks:

```bash
cat /workspace/cactus_cq_runs/MODEL_L4/conversion_summary.json
cat /workspace/cactus_cq_runs/MODEL_L4/hessian_metadata.json
```

Look for:

- `unrecognized` should be `0`.
- GPTQ targets should have nonzero `hessian_samples`.
- `errors` in `hessian_metadata.json` should be empty.
- Fallbacks should be expected tensors such as norms, positional embeddings, or supported INT8 biases/convs.

## 5. QDQ For Validation

Convert a Cactus directory back into a PyTorch/HF-style checkpoint:

```bash
python -m src.cq_convert qdq \
  /workspace/cactus_cq_runs/MODEL_L4 \
  --out /workspace/cactus_cq_runs/MODEL_L4_qdq_fp16 \
  --model-family auto \
  --force
```

Validate tensor coverage against the source model:

```bash
python -m src.cq_convert validate \
  --source-model /path/to/hf_model \
  --qdq /workspace/cactus_cq_runs/MODEL_L4_qdq_fp16 \
  --model-family auto
```

## 6. Package For Cactus

The runtime package is a zip of the conversion directory. Exclude Hessian files.

```bash
python - <<'PY'
from pathlib import Path
import zipfile

src = Path("/workspace/cactus_cq_runs/MODEL_L4")
dst = Path("/workspace/cactus_cq_runs/hf_upload/MODEL/L4.zip")
dst.parent.mkdir(parents=True, exist_ok=True)

with zipfile.ZipFile(dst, "w", compression=zipfile.ZIP_DEFLATED, compresslevel=6) as zf:
    for p in sorted(src.iterdir()):
        if not p.is_file():
            continue
        if p.name == "hessians.safetensors":
            continue
        info = zipfile.ZipInfo(p.name)
        info.date_time = (1980, 1, 1, 0, 0, 0)
        info.compress_type = zipfile.ZIP_DEFLATED
        zf.writestr(info, p.read_bytes())

print(dst, dst.stat().st_size)
PY
```

Packages must be self-contained. They should include:

- `config.txt`
- all required `.weights` files
- tokenizer files such as `tokenizer.json`, `tokenizer_config.txt`,
  `special_tokens.json`, `vocab.txt`, `merges.txt`
- `chat_template.jinja2` when the model uses one

Do not include `hessians.safetensors`.

## 7. Upload To Hugging Face

```bash
python - <<'PY'
from pathlib import Path
from huggingface_hub import HfApi, create_repo

repo = "Cactus-Compute/MODEL-cq"
root = Path("/workspace/cactus_cq_runs/hf_upload/MODEL")

create_repo(repo, repo_type="model", exist_ok=True)
api = HfApi()

for name in ["L1.zip", "L2.zip", "L3.zip", "L4.zip"]:
    path = root / name
    if path.exists():
        print("uploading", name, path.stat().st_size)
        api.upload_file(
            path_or_fileobj=str(path),
            path_in_repo=name,
            repo_id=repo,
            repo_type="model",
            commit_message=f"Upload {name}",
        )
PY
```

For VLM packages, upload `L1V4.zip`, `L2V4.zip`, `L3V4.zip`, and `L4V4.zip`
instead.

## Practical Notes

- Use `--model-family auto` unless auto-detection picks the wrong adapter.
- Use explicit `--model-family qwen`, `lfm2`, `gemma4`, `whisper`, `parakeet`,
  or `parakeet_tdt` when debugging a known family.
- Text-only models should not include vision calibration data.
- A model with no vision tower should not use `V` package names.
- Embeddings normally remain CQ4 even in `L1`, `L2`, and `L3` packages.
- Norm tensors currently remain FP16.
- Bias tensors generally use INT8 where the runtime format supports it.
- Keep source code in `python/src/cq_convert`; keep generated artifacts
  in `/workspace/cactus_cq_runs`.
